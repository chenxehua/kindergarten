## 1. 项目概述

### 1.1 项目背景

随着家庭教育观念的升级和家长对孩子在园生活关注度的提高，幼儿园亟需一套数字化、系统化的学生成长管理平台。该平台旨在实现幼儿园日常管理的智能化，帮助老师高效记录学生在校情况，同时为家长提供孩子成长的实时了解渠道。通过AI技术的引入，系统能够基于历史数据为每位学生生成个性化的成长画像和月度总结，并自动生成成长视频，让每个孩子的成长轨迹可视化、可追溯、可分享。

### 1.2 项目目标

本系统旨在构建一个家园共育的数字化平台，实现以下核心目标：第一，建立完整的学生信息档案库，实现学生、班级、老师、家长的统一管理；第二，支持老师每日便捷记录学生在校的课程学习、情绪状态、饮食情况、户外活动等动态；第三，引入AI能力，基于长期积累的数据为每个学生生成成长画像和月度成长报告；第四，利用AI生成技术和多媒体处理能力，自动将照片素材合成为个性化的成长视频；第五，提供多角色登录入口，区分家长、老师、园长的功能权限和数据可见范围。

## 2. 用户角色与权限体系

### 2.0 登录方式说明

| 角色 | 登录入口 | 终端 |
|------|---------|------|
| 家长 | 微信小程序 | 手机/平板 |
| 老师 | 微信小程序 | 手机/平板 |
| 园长 | PC管理后台 | 电脑浏览器 |

### 2.1 用户角色定义

系统涉及四类核心用户角色，每类角色拥有独立的功能模块入口和数据访问权限。

**家长（微信小程序）**是系统的核心用户群体，每位学生可关联多名家长账号（爸爸、妈妈、爷爷、奶奶等）。小程序端设计遵循轻量化、高频交互原则，主要功能包括：
- 查看自己孩子的全部成长记录、在校情况
- 接收通知公告、园所活动报名
- 与老师进行即时沟通（文字/语音/图片）
- 查看孩子的AI成长画像、月度成长报告
- 查看、下载、分享成长视频
- 接收每日推送（可设置免打扰时段）
- 完善孩子和家庭的基础信息

**老师（微信小程序）**是日常记录的主要操作者，小程序端设计注重快捷录入、单手操作，主要功能包括：
- 每日成长记录发布（支持语音输入、AI辅助填写）
- 拍照上传活动照片（支持批量上传、AI美化）
- 课程安排查看与签到
- 所负责班级学生信息管理
- 与家长一对一沟通
- 查看班级整体情况统计
- 食谱查看（了解今日餐食）
- 考勤打卡

**园长（PC管理后台）**是系统管理员角色，拥有最高级别的数据访问权限，PC端设计注重效率与批量操作，主要功能包括：
- 全园学生、老师、班级信息管理
- 所有班级的日常记录统计和数据分析报表
- 课程体系管理、食谱安排制定
- 通知公告发布（可群发/定向发送）
- 运营数据看板（出勤率、家长活跃度、教学质量等）
- 成长报告审核与批量导出
- 视频模板配置与审核发布
- 系统设置与权限配置

**系统管理员**负责系统的后台运维，通过PC管理后台进行用户权限配置、系统参数设置、数据备份与恢复等技术支持工作。

## 3. 功能模块详细需求

### 3.1 学生信息管理

学生信息管理是系统的基础模块，用于维护全园学生的完整档案。每个学生的信息记录应包含以下内容：

**基本信息**：
- 学生姓名（必填）
- 性别（必填）
- 出生日期（必填）
- 身份证号（可选，用于学籍）
- 头像照片（必填，支持拍照或相册选择）
- 入园日期（必填）
- 所属班级（必填，关联班级管理）
- 学号/在园编号（系统自动生成）

**健康信息**：
- 过敏史（多选：牛奶、鸡蛋、海鲜、花生等，可自定义添加）
- 既往病史（文字描述：先天性心脏病、哮喘等）
- 身高体重发育曲线（定期记录，关联生长曲线图）
- 预防接种记录（可上传接种本照片）
- 医保信息（医保卡号、绑定医疗机构）

**家庭信息**：
- 家庭住址（省市区+详细地址）
- 紧急联系人（姓名、电话、与学生关系）
- 家长联系方式（至少一个主要联系人）
- 家庭成员构成（可选）

**成长档案**：
- 成长照片墙（按时间轴展示）
- 关键里程碑记录（第一次入园、第一次获奖等）
- 成长大事记（老师可添加）

**扩展信息**：
- 兴趣爱好（多选+自定义）
- 特长能力（文字描述）
- 性格特点（多选标签）

**系统应支持**：
- 学生信息的增删改查操作
- 学生转班或离园时支持状态变更（转班/离园/休学），历史数据应保持可追溯
- 老师录入学生信息时应支持批量导入功能（Excel模板导入）
- 家长可通过移动端自主完善部分信息（如家庭信息更新）
- 学生信息变更需记录操作日志和变更时间
- 支持学生信息导出（Excel格式）
- 支持按班级、状态、时间段等条件筛选学生

#### 3.1.1 信息录入流程

新生入园流程：园长或老师录入基本信息 → 上传证件照 → 录入健康信息 → 关联家长账号 → 分配班级 → 完成入园

信息更新流程：信息变更申请 → 老师或家长提交 → 园长审批（如需） → 生效

#### 3.1.2 批量操作

- 支持学生批量导入（新生入园高峰期）
- 支持学生批量转班
- 支持学生批量离园处理
- 支持批量更新班级信息

### 3.2 班级管理

班级管理模块用于维护幼儿园的班级架构体系，支持多园区、多班级管理。

**班级基本信息**：
- 班级名称（如星星班、阳光班、向日葵班）
- 班级编号（系统唯一标识）
- 所属年级（小班/中班/大班/学前班）
- 所属园区（如有多园区需区分）
- 教室位置（楼栋+楼层+房间号）
- 班主任（必填，关联老师信息）
- 副班老师（可多个）
- 保育员（可多个）
- 在班学生人数（实时统计）
- 班级容量（最大人数限制）
- 班级状态（在班/已结班/待开班）

**班级详细信息**：
- 班级简介（文字描述）
- 班级公约（班级规则）
- 班级相册（班级活动照片）
- 班级荣誉（获得的奖项）
- 班级日志（班级记事）

**班级页面展示**：
- 当日出勤率统计（出勤/请假/缺勤）
- 今日课程安排（当日课程表）
- 近期活动回顾（近7天）
- 班级通知（未读消息数）
- 今日食谱（对接食谱模块）

**系统应支持**：
- 灵活的班级设置操作
- 升班处理（学期结束时批量升级）
- 并班处理（两个班级合并为一个）
- 转班处理（学生从A班转至B班）
- 班级学期切换（新学期新班级）
- 班级历史数据查询（已结班级的历史记录）

#### 3.2.1 升班处理

- 每学年结束执行升班操作
- 自动更新学生年级信息
- 自动创建新班级或继承原班级
- 支持升班预演（查看影响范围）
- 升班前自动生成班级报告

#### 3.2.2 并班/分班处理

- 并班前需确认学生分配方案
- 并班后保留双方历史记录
- 分班支持按学生或按老师拆分
- 操作需园长审批

#### 3.2.3 班级看板

班级首页应展示：
- 班级人数统计（男/女/总计）
- 本月出勤率趋势图
- 本月成长记录发布统计
- 家长互动活跃度（消息数/回复率）
- 班级课程完成进度
- 近期需要关注的事项提醒

### 3.3 老师管理

老师管理模块用于维护幼儿园教职工信息。

**老师基本信息**：
- 姓名（必填）
- 工号（系统唯一标识）
- 职位类型（主班老师/副班老师/保育员/保健医/行政人员/后勤人员/园长/副园长）
- 所属部门/园区
- 联系电话（必填）
- 紧急联系人电话
- 入职日期（必填）
- 头像照片
- 学历背景（可选）
- 专业背景（可选）
- 资质证书（教师资格证、育婴师证等，可上传证件照）
- 个人简介

**工作安排**：
- 所负责班级（可多个）
- 排课信息（每周课时安排）
- 考勤信息（每日签到签退）
- 工作日志

**系统应支持**：
- 老师信息的增删改查
- 老师排课和考勤管理
- 记录每位老师每日的工作安排
- 老师可查看自己在园所的班级和工作任务
- 支持调课、代课等临时变更处理
- 老师离职/调岗处理（保留历史记录）
- 老师工作量统计

#### 3.3.1 考勤管理

- 上班签到（位置打卡+人脸识别，可选）
- 下班签退
- 请假申请（年假/病假/事假）
- 代课安排
- 考勤统计报表

#### 3.3.2 权限管理

- 按角色分配功能权限
- 按班级分配数据可见范围
- 临时权限授予（如代班老师）

### 3.4 家长管理

家长管理模块实现家长账号的注册、认证和关联。

**家长基本信息**：
- 姓名（必填）
- 与学生关系（爸爸/妈妈/爷爷/奶奶/外公/外婆/其他）
- 联系电话（必填，用于登录）
- 微信openid（微信小程序登录必填）
- 头像照片
- 身份证号（可选，用于实名认证）

**通知偏好设置**：
- 接收通知方式（推送/短信/两者都要）
- 免打扰时段设置（可设置多个时间段）
- 接收通知类型选择（成长记录/通知公告/活动报名/考勤提醒等）
- 语言设置（中文/英文）

**账号安全**：
- 登录密码（可设置）
- 支付密码（用于付费功能，可选）
- 绑定手机号
- 微信绑定状态

**关联学生**：
- 关联的学生（可多个，需通过幼儿园审核）
- 关联关系（如：孩子爸爸、孩子妈妈）
- 主账号标识（每个学生可设置一个主账号）

**系统应支持**：
- 一个学生关联多个家长账号
- 家长之间可互相查看孩子的动态（可选配置，默认关闭）
- 家长账号支持手机号验证码登录
- 家长账号支持微信授权登录
- 家长账号注销申请（需园长审批）
- 家长换绑手机号

#### 3.4.1 账号注册与认证

**微信小程序登录流程**：
1. 微信授权获取用户信息
2. 输入手机号获取验证码
3. 验证成功后关联学生信息
4. 完成注册

**账号激活**：
- 新生家长由老师手动关联
- 家长可自主申请关联学生（需园长审批）

#### 3.4.2 家长端功能限制

- 家长只能查看自己孩子的信息
- 家长不能修改孩子的基础信息（需通过老师）
- 家长不能查看其他孩子的信息
- 家长不能进行批量操作

### 3.5 食谱管理

食谱管理模块用于幼儿园的营养膳食计划管理。

**每周食谱结构**：
- 周期选择（按周设置，每周日开始，周六结束）
- 适用园区/班级（全园/指定班级）
- 食谱状态（草稿/已发布）

**每日食谱内容**：
- 早餐（主食品种+饮品+水果，可多个菜品）
- 上午点心（时间+食品）
- 午餐（主食+菜品+汤品，可多个菜品）
- 下午点心（时间+食品）
- 晚餐（针对寄宿制幼儿园）

**每道菜品信息**：
- 菜品名称（必填）
- 菜品图片（可选）
- 主要食材（列出主要原料）
- 营养成分（热量/蛋白质/脂肪/碳水化合物/维生素等，可引用标准数据）
- 过敏原提示（标记可能引起过敏的成分）
- 适宜人群（根据年龄适配）
- 特殊说明（如：少油少盐等）

**营养分析**：
- 每日营养摄入统计（与儿童膳食指南对比）
- 每周营养均衡分析
- 营养达标提醒（如钙摄入不足时提示）
- 针对过敏学生的提醒（自动过滤含过敏原的餐点）

**系统应支持**：
- 营养师或园医根据儿童膳食指南制定每周食谱
- 食谱应提前一周发布并通知家长
- 食谱模块应与学生健康档案联动，对于有食物过敏的学生应自动在相关餐点时发出提醒
- 食材库存管理（可选）
- 采购计划生成（可选）
- 每日消耗统计（可选）
- 食谱复制功能（可复制上周食谱）
- 食谱模板功能（常用食谱保存为模板）

#### 3.5.1 食谱发布流程

1. 营养师制定下周食谱（可使用模板）
2. 园医审核营养搭配
3. 园长审批通过
4. 定时发布（可设置发布后通知家长）
5. 家长端可见

#### 3.5.2 家长端展示

- 按日期查看食谱
- 订阅提醒（每日食谱推送）
- 查看营养成分详情
- 餐食反馈（可选择"喜欢"或"不喜欢"）
- 过敏提醒（自己孩子有过敏原时自动高亮）

### 3.6 课程管理

课程管理模块用于规划和记录幼儿园的教学活动。

**课程体系**：
- 课程名称（必填，如：创意美术、趣味数学、音乐启蒙等）
- 课程编号（系统唯一标识）
- 课程类别（艺术/科学/语言/运动/社会/健康/数学/英语）
- 适用年龄（小班/中班/大班/学前班）
- 授课老师（可多个，需具备相应资质）
- 课时时长（如：30分钟/45分钟）
- 课程目标（教学目标描述）
- 教学大纲（课程内容概要）
- 所需材料（教具、材料清单）

### 3.8 AI成长画像（核心功能）

AI成长画像模块是本系统的智能化核心，基于日常积累的数据自动分析每位学生的成长特征和发展趋势。

#### 3.8.1 画像维度细分

成长画像应包含以下分析维度：

**情绪画像**：
- 情绪稳定性评估（波动频率、幅度）
- 情绪表达方式（语言表达/肢体表达/表情表达）
- 高频情绪类型（开心占比/平静占比/低落占比等）
- 情绪变化规律（时间段规律/事件触发规律）
- 与同龄人对比的情绪发展水平
- 情绪预警提醒（如连续低落超过3天）

**社交画像**：
- 社交主动性评分（主动发起互动频率）
- 同伴交往能力（朋友数量、友谊稳定性）
- 合作分享表现（愿意分享/合作完成任务）
- 冲突解决方式（求助老师/自行解决/哭闹）
- 社交行为分析（积极行为/消极行为比例）

**学习画像**：
- 课程参与度（每次课程参与情况）
- 学习兴趣倾向（哪类课程最感兴趣）
- 专注力表现（持续专注时长、注意力分散情况）
- 知识吸收能力（掌握新知识的速度）
- 学习习惯（倾听习惯、动手操作习惯）
- 课堂表现评级

**运动画像**：
- 运动能力发展（大运动/精细运动）
- 体能变化趋势（耐力/力量/柔韧性）
- 运动兴趣偏好（喜欢哪类运动）
- 运动技能掌握（跳跃/跑步/攀爬/投掷等）
- 运动参与积极性

**饮食画像**：
- 饮食规律（用餐时间是否规律）
- 营养摄入均衡性（各类食物摄入比例）
- 饮食习惯改善情况（偏食改善进度）
- 饮食量变化（食欲波动）
- 特殊饮食需求管理

**发展预警**：
- 情绪持续低落超过预警阈值
- 食欲明显下降
- 社交退缩表现
- 发育偏离正常曲线
- 行为异常信号检测
- 专业建议生成（根据预警级别）

#### 3.8.2 画像数据来源

画像分析数据来源：
- 每日成长记录（情绪、饮食、活动等）
- 课程参与记录和表现评价
- 照片和视频中的行为分析（可选，需授权）
- 家长反馈信息
- 阶段性测评数据

#### 3.8.3 画像展示

**可视化呈现**：
- 雷达图（各维度发展情况对比）
- 趋势图（各维度随时间变化）
- 发展曲线（身高体重等生理指标）
- 进度条（目标达成情况）

**历史对比**：
- 按时间维度查看历史变化
- 月度/季度/学期对比
- 与同龄人平均水平对比

**报告内容**：
- 各维度详细分析解读
- 发展优势分析
- 需要关注事项
- 针对性教育建议（可打印）
- 专家建议（可预约专家咨询）

#### 3.8.4 画像更新机制

- 每日更新基础数据
- 每周生成周度小结
- 每月生成本月画像
- 季度/学期生成综合报告
- 画像更新推送通知家长

#### 3.8.5 隐私与权限

- 画像数据仅家长和负责老师可见
- 园长可见班级汇总画像（脱敏）
- 画像数据归属权说明
- 家长可选择关闭AI分析功能

### 3.9 月度成长报告（核心功能）

系统应每月自动生成每位学生的成长报告，作为家园沟通的重要载体。

#### 3.9.1 报告内容详细

月度成长报告应包含以下内容：

**报告基本信息**：
- 报告周期（2024年3月）

- 与上月对比（如：出勤率上升5%）

**成长亮点**：
- 本月最值得记录的成长瞬间（3-5条，从日常记录中自动提取）
- 本月进步最大的方面
- 本月获得的荣誉或成就
- 老师特别表扬的内容

**各维度发展情况**：
| 维度 | 发展评级 | 本月表现 | 与上月对比 | 详细描述 |
|------|---------|---------|-----------|---------|
| 情绪 | ⭐⭐⭐⭐⭐ | 情绪稳定，阳光开朗 | ↑ 提升 | ... |
| 社交 | ⭐⭐⭐⭐ | 主动友好，喜欢分享 | → 持平 | ... |

- 家长可预约老师沟通时间

### 3.10 AI成长视频（核心功能）

成长视频是本系统的亮点功能，利用AI技术自动将照片素材合成为个性化的成长视频。

#### 3.10.1 视频生成模式

**月度成长视频**：
- 生成时间：每月初自动生成上月视频（如4月1日生成3月视频）
- 照片选取：选取本月最精彩的20-30张照片
- 视频时长：1-3分钟（可选择）
- 生成后推送给对应家长

**成长回顾视频**：
- 时间范围：按学期（上半年/下半年）、按学年
- 照片数量：50-100张照片
- 视频时长：3-5分钟
- 支持手动触发生成

**精彩瞬间视频**：
- 触发条件：特定活动后（如运动会、文艺汇演、亲子活动、春游秋游）
- 自动生成：活动结束后自动生成
- 照片来源：活动中拍摄的照片
- 视频时长：30秒-1分钟

#### 3.10.2 视频生成技术

**智能选片**：
- 基于照片质量（清晰度、构图）
- 基于照片内容（人物表情、场景多样性）
- 基于照片时间分布（覆盖整月）
- 基于照片人物（确保每个幼儿都有出镜）

**智能剪辑**：
- 自动转场特效（渐变、滑动、缩放等）
- 背景音乐自动匹配（节奏适配）
- 照片停留时间自动调整（根据内容重要程度）
- 片头片尾自动生成

**视频效果**：
- 支持多种视频模板（卡通/温馨/活泼/简约）
- 支持滤镜效果
- 支持添加文字字幕
- 支持配音旁白（AI生成，可选）

#### 3.10.3 视频定制能力

**模板选择**：
- 内置多种视频模板
- 模板风格：卡通动漫/实景拍摄/亲子风格/节日主题
- 园长可配置默认模板
- 家长可选择偏好模板

**音乐定制**：
- 可替换背景音乐
- 内置儿歌/轻音乐库
- 支持上传本地音乐
- 音量可调节

**文字定制**：
- 可添加文字字幕
- 支持添加学生姓名
- 支持添加日期

## 4. 非功能性需求

### 4.1 性能需求

系统应满足以下性能指标：

**响应时间**：
- 核心页面（首页、记录列表）加载时间不超过3秒
- 高并发访问时响应时间不超过5秒
- 视频生成任务在30分钟内完成（如需加急可走付费通道）
- 图片上传到可预览不超过5秒

**并发能力**：
- 支持至少500个并发用户同时在线
- 支持100个视频生成任务同时排队处理
- 支持照片批量上传（100张）不超时

**系统可用性**：
- 系统全年可用性不低于99.9%
- 计划内维护窗口：每月一次，凌晨2:00-6:00
- 核心服务故障自动切换时间<30秒

**存储容量**：
- 单园所支持存储100GB照片
- 单视频最大支持500MB
- 支持存储空间扩展

### 4.2 安全需求

**数据安全**：
- 所有用户数据应加密存储（AES-256）
- 所有数据传输应加密（HTTPS/TLS 1.3）
- 敏感信息（身份证号、联系方式）脱敏显示
- 数据备份每日自动执行，保留30天

**访问控制**：
- 家长查看孩子信息需进行身份验证（登录态校验）
- 角色权限严格分离（RBAC模型）
- 操作日志完整记录（保留180天）
- 敏感操作需二次验证（如删除数据）

**存储安全**：
- 照片和视频应存储在安全的云存储服务（OSS加密存储）
- 云存储开启防盗链
- 视频文件不支持直接URL下载

**隐私保护**：
- 人脸识别数据仅本地处理，不上传云端（可选）
- 家长可选择关闭AI分析功能
- 家长可申请导出/删除个人数据
- 未成年人信息保护符合法规要求

**审计追溯**：
- 完整的操作日志审计功能
- 日志内容包括：操作人、操作时间、操作内容、IP地址
- 异常操作自动告警

### 4.3 兼容性需求

**移动端兼容性**：
- iOS 12及以上版本
- Android 8及以上版本
- 微信小程序基础库2.20以上版本
- 移动端应适配各种主流屏幕尺寸（手机/平板横竖屏）

**PC端兼容性**：
- Chrome 90及以上版本
- Edge 90及以上版本
- Safari 14及以上版本
- Firefox 88及以上版本
- 屏幕分辨率支持：1366x768 / 1920x1080 / 2560x1440

**浏览器特性**：
- 支持HTML5视频播放
- 支持WebSocket通信
- 支持本地存储
- 禁用低版本浏览器访问提示

### 4.4 扩展性需求

**功能扩展**：

## 5. 技术架构建议

### 5.1 移动端技术栈

移动端APP建议采用Flutter或React Native实现跨平台开发，确保iOS和Android体验一致。微信小程序作为重要的入口载体，需进行专门适配。

### 5.2 后端技术栈

后端服务建议采用微服务架构，主要技术选型包括：Spring Cloud作为微服务框架；MySQL作为主数据库存储业务数据；MongoDB存储非结构化的日志和原始数据；Redis用于缓存和会话管理；对象存储服务（如OSS/MinIO）存储照片和视频文件。

### 5.3 AI能力

AI能力建议通过调用云服务实现：AI成长画像可调用大语言模型进行语义分析和成长评估；视频生成可调用视频AI服务进行智能剪辑和合成；图像处理可调用云端AI进行智能修图和人像优化。

### 5.4 部署架构

系统建议采用云原生部署方式：使用Kubernetes进行容器编排和服务治理；使用CDN加速静态资源和视频分发；核心数据库采用主从复制保证高可用；视频处理任务使用异步队列解耦。

## 6. 实施里程碑

### 阶段一：基础功能建设（第1-3个月）

完成用户体系搭建、学生管理、班级管理、老师管理、家长管理的基础CRUD功能；完成食谱管理和课程管理的基础功能；完成每日成长记录的文字记录功能（不含AI分析）。

### 阶段二：核心功能上线（第4-6个月）

上线每日成长记录的照片上传功能；开发AI成长画像模块并上线；上线月度成长报告功能；完成移动端APP和小程序的开发和上线。

## 9. 接口设计

### 9.1 接口设计原则

- RESTful风格，URL语义清晰
- 采用JSON格式传输数据
- 版本号管理（如：/api/v1/）
- 统一响应格式
- 请求需要身份验证（Token）
- 接口文档使用OpenAPI 3.0规范

### 9.2 统一响应格式

**成功响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": 1699999999999
}
```

**错误响应**：
```json
{
  "code": 1001,
  "message": "参数错误",
  "timestamp": 1699999999999
}
```

**分页响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [ ... ],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 100,
      "totalPages": 5
    }
  },
  "timestamp": 1699999999999
}
```

### 9.3 错误码定义

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1000 | 系统错误 |
| 1001 | 参数错误 |
| 1002 | 权限不足 |
| 1003 | 资源不存在 |
| 1004 | 业务逻辑错误 |
| 1005 | 请求超时 |
| 2001 | 用户不存在 |
| 2002 | 密码错误 |
| 2003 | 验证码错误 |
| 2004 | Token过期 |
| 2005 | 用户禁用 |
| 3001 | 学生不存在 |
| 3002 | 班级不存在 |
| 4001 | 文件上传失败 |
| 4002 | 文件格式不支持 |

## 10. 数据库设计

### 10.1 数据库设计原则

- 遵循第三范式（3NF）
- 表名使用下划线命名法
- 字段名使用下划线命名法
- 每张表必须有主键
- 必须包含创建时间和更新时间字段
- 软删除优先（使用status字段）
